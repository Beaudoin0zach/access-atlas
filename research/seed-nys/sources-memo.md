# Sources Memo — WNY Accessibility Seed Data for Access Atlas

**Prepared:** 2026-07-08 · **Scope focus:** Western New York (Buffalo / Erie County), per §3 beachhead strategy · **Purpose:** annotated, verified inventory of registries/datasets to mine for seed data. This is a *lead-generation* memo, not a data pull. Every URL below was fetched or search-confirmed during research; unverifiable claims and coverage gaps are flagged explicitly.

> **Labeling reminder (§4).** No source below produces a trustworthy label on its own. Everything ingested from these sources enters Access Atlas as `self_reported` and earns a higher state only through our own community consensus (≥3 first-person confirmations) or a real certification/audit (`sourced`). Third-party confidence never transfers.

## Sources we should NOT copy from (and why)

- **AXS Map — the single overall star rating.** It collapses accessibility into one 1–5 star average, the exact failure Access Atlas rejects (§4 attribute-level; §14 "do not collapse into a single overall star"). Fine as a *list of venues to investigate*; its ratings must never become our labels.
- **Google Maps / Yelp / AccessNow attributes as imported truth.** These are (a) self-reported by owners or coarsely crowd-sourced — not first-person, photo-backed, ≥3-consensus (§4) — and (b) ToS-restricted against redistribution/caching. Using them as our labels would violate both our honest-labeling non-negotiable and likely their terms. Lead sources only.
- **Any external rating as a `verified` / `high confidence` label.** Per §4/§14, only our own community-consensus flow yields `community-verified`, and only certifications/audits yield `sourced`. No third party's confidence transfers.
- **Stale/undated crowdsourced dumps.** Wheelmap/OSM data can be years old; physical access decays (§4 time-decay). Ingest as *candidates to re-verify*, never as current fact, and attach our own "last confirmed" date.

## Highest-value first pulls (mine these first for WNY density)

1. **NYAIL/NYSILC + ACL CIL list → WNYIL and the WNY ILC cluster.** Small, authoritative, directly actionable. Anchors the §3 partner strategy and gives credible Provider seeds plus reviewer-recruitment channels. Start here. *(Already captured in `listings.json`.)*
2. **NFTA Metro GTFS (static zip).** The single richest *first-party* WNY dataset with real accessibility signal (accessible stops/stations). Bulk-loadable as Places. (Watch the SSL cert quirk.) *(Four rail stations captured manually; the GTFS pull would add every stop.)*
3. **NYS SDVOB directory, filtered to Erie County.** Official state certification = high-reliability disabled-owned (veteran subset) seeds. Needs scraping or an OGS data request; scope the "disabled-owned" tag honestly.
4. **OPWDD service-provider directories (data.ny.gov, Socrata API) filtered to Erie/WNY.** Clean CSV/API, county-coded, disability-serving organizations → strong disability-literate **Provider** candidate list.
5. **Open Data Buffalo Business Licenses + Wheelmap OSM POIs (WNY bbox).** Two complementary *base lists of venues to investigate* — the license file for breadth of real businesses, Wheelmap for the subset already flagged wheelchair-relevant — then route both through Access Atlas's own attribute-level, photo-backed validation.

## Documented gaps (valid findings — see also `gaps.md`)

- **No curb-ramp / accessible-parking / 311 dataset on Open Data Buffalo.** Confirmed via the Socrata catalog API against Buffalo's own domain — the promising `q=accessible` hits are federated results from *other* cities' portals.
- **No statewide accessible-parking or curb-ramp dataset on data.ny.gov.** ADA fields there are essentially NYC/MTA-scoped and don't help WNY.
- **Buffalo GIS curb-cut layer is unconfirmed as published** — it exists in-migration per city RFP docs but no downloadable public layer was verified. A human should probe the ArcGIS REST directory.
- **No national/state registry of MDE-compliant providers exists.** The July 8 / Aug 9 2026 deadlines create the obligation but not a lookup list — precisely the whitespace Access Atlas's community validation fills. Use the deadlines as a recruitment hook, not a data source.
- **General accessibility directories (AXS Map, AccessNow) have thin, unverified WNY coverage** — consistent with §3's premise that thin data is the platform-killer. Density will come from our own seeding, not from importing theirs.
